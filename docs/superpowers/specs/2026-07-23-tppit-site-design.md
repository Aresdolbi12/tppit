# tppit.ru — современный сайт ТПП Краснодарского края (спека)

Дата: 2026-07-23. Статус: одобрено пользователем (Владислав, ОИТ ТПП КК).

## ПЕРЕСМОТР БАЗЫ 2026-07-24 (подтверждён)

- **Роль:** tppit.ru — будущий ОСНОВНОЙ сайт палаты. kuban.tpprf.ru контролирует
  федеральная ТПП РФ (Bitrix, доступ уровня «добавить новость»; новая
  newkuban.tpprf.ru — тоже Bitrix без админ-доступа). Переход палаты на свой сайт.
- **CMS:** обязательна, готовая удобная админка для сотрудников (кандидаты:
  Directus, Strapi — выбор на этапе внедрения).
- **Хостинг:** VPS с Docker (провайдер решается перед деплоем; кандидат reg.ru).
  Этап «PHP API на DELENERGO» ОТМЕНЁН.
- **Рендеринг:** при внедрении CMS — переезд на Next.js (SEO, чистые URL, OG).
- **Порядок:** довести демо на Pages → показ руководству → добро на бюджет
  и переход → VPS + CMS + Next.js + миграция реестра (дамп MySQL) + ЛК.
- **Главная:** новости с картинками на первом месте (по образцу tpprf.ru) —
  публичная деятельность и есть журнал «Вестник ТПП» (PDF-архив за 5 лет
  у пользователя в D:\работа\ТПП\Вестник).

## Цель

tppit.ru строится как **полноценный современный сайт палаты** — по наполнению повторяет
и превосходит kuban.tpprf.ru, в перспективе способен его заменить. Штрих-коды — одна из
услуг (отдельный раздел), а не тема всего сайта. Контент — только реальные факты
(источник: исследование `tppkuban/raw/kuban-tpprf-ru-content-2026-07-23.md` в vault).

## Архитектура

```
web/            React SPA (публичный сайт + /admin + ЛК)
api/            PHP 8.2: роутер + PDO + JSON (на DELENERGO в /api/*)
  /api/check    проверка кода по реестру (code | inn)
  /api/apply    приём заявок (услуги, членство, штрих-коды) → почта отдела + MySQL
  /api/auth     сессии для админки и ЛК (email-код / пароль)
  /api/content  CRUD контента для админки (услуги, люди, тексты, тарифы)
  /api/lk/*     ЛК: заявки на коды, статусы, файлы макетов/свидетельств
MySQL           delenergo_tppitru: реестр codes (существует) + новые таблицы
                (content, applications, users, lk_orders, lk_files)
```

- Демо-режим на Pages: те же страницы, данные из ts/json-файлов, API замокан.
- Секреты только на сервере (`api/secrets.php`, не в репо).
- Спам-защита форм: honeypot + время заполнения + rate limit по IP.

## Этапы

1. **Контент** (Pages): все разделы со статичным реальным контентом; правки через репо
2. **API + бой**: PHP API (check, apply), деплой на tppit.ru (docs_new → swap, PHP 8.2, HTTPS-сертификат уже выпущен до 2026-10-20)
3. **Админка**: контент в MySQL, /admin, роли
4. **ЛК штрих-кодов**: регистрация, подача списка продукции, статусы, файлы
5. **Постоянно**: CI-обновление новостей/анонсов, продление SSL (задача в vault)

## Риски

- DELENERGO слаб/ограничен → план Б: VPS, фронт без изменений
- Расхождение контента двух сайтов → глубокие регламенты ссылками, пока не решено о замене
- Спам в формах → honeypot + rate limit; почтовая отправка через mail() хостинга, при проблемах — SMTP
- «32 палаты» и «5 200+ предприятий» со старого tppit.ru устарели — использовать данные основного сайта (11 межрайонных ТПП и т.д.)
